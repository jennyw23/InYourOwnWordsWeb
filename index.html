In Your Own Words

We provide a public dataset and computational framework for automatically discovering interpretable themes from free-text survey data. Applied to 1,004 U.S. participants' descriptions of race, gender, and sexual orientation, our method surfaces themes — like identity fluidity, belonging, and cultural pride — that standard survey categories miss. These themes reveal heterogeneity within demographic groups and illuminate how self-identified identities diverge from perceived ones.

Methodology
The Analysis Pipeline

We combine large-scale text embeddings with a Sparse Autoencoder (SAE) and LLM interpretation to automatically discover interpretable identity themes from free-text survey responses.

🔢
text → 3,072-dim vector
Compute Text Embeddings

Each free-text response is encoded into a dense semantic vector using OpenAI's text-embedding-3-large model, producing a 3,072-dimensional representation that captures the semantic content of the response.

⚡
3,072-dim → 32 features, K=4 active
Train Sparse Autoencoder (SAE)

A Sparse Autoencoder (SAE) is trained separately on each identity's embeddings. The SAE learns a sparse, overcomplete dictionary of M=32 features, where at most K=4 features are active for any given response. This sparsity constraint encourages each feature to capture a distinct, interpretable aspect of identity description.

💬
Feature activations → human-readable theme label
LLM-Interpret Each Feature

The top-activating responses for each SAE feature are passed to GPT-4o, which generates a natural-language description of what the feature represents. Features with poor fidelity (annotation F1 < 0.50) are excluded.

🏷️
All responses × all themes → binary annotations
Annotate All Responses for Every Theme

Each theme label is used as a prompt for GPT-4.1-mini to annotate every response (0/1) — not just the top-activating ones. This produces a full binary annotation matrix for downstream analysis, validated against human raters (κ = 0.65).
